# Security Connection

** Цель работы - реализовать защищенное соединение между клиентом и сервером **
Использованые методы для поставленной задачи: 
  V1.0
  алгоритм Diffie–Hellman, для создания общего секретного ключа и дальнейшего его использования для шифрования симметричными алгоритмами 
  блочный алгоритм симметричного шифрования AES 
  HMAC (hash-based message authentication code)
  Flask
  
  V2.0 ( in process ) 
  OpenSSL

В первой версии реализовано шифрование передаваемых данных на прикладном уровне модели TCP/IP.
Но шифрование данных на прикладном уровне еще не обеспечивает безопасность, так как ключи
во время установления соединения передаются по открытому каналу связи по протоколу TCP, это происходит на
транспортном уровне.
Поэтому вскоре будет добавлен протокол TLS/SSL для защиты передаваеиых данных ( TLS Handshake ) 
для этого я с помощью алгоритмов шифрования сгенерирую приватный ключ, и им подпишу сертификат 
сделанный с использованием OpenSSL
